Benchmarking and verification of Julius

In order to verify the correctness and speed of the implementations in Julius, we compare ourselves to different reference implementations, comparing speed and checking how far we are.

ResampleFrac

We compare julius.resample to resampy, on an input of size (32, 8 * 44100), i.e. a batch of size 16 of 8 second of audio at 44.1kHz. We use the same number of zero crossing as resampy for this benchmark. The small delta is probably due to the different window function used.

On CPU we have:

Old sr New sr Julius (ms) Resampy (ms) Delta (%)
2 1 288 1530 1.3%
1 2 367 2329 1.8%
4 5 91 1439 1.8%
10 11 54 1260 1.8%
44100 16000 62 2180 1.0%
20001 30001 19922 2313 1.8%
